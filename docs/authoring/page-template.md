# ページ作成テンプレート

新しい「やりたいこと」ページを追加するときのひな型です。

````markdown
# やりたいこと

何をする操作かを 1〜2 文で説明します。

## レーンの動作

```text
入力 A: ...
入力 B: ...
結果:   ...
```

## 関連する intrinsic

| 操作 | intrinsic | 公式ドキュメント |
|---|---|---|
| ... | `_mm...` | [Intel](https://www.intel.com/content/www/us/en/docs/intrinsics-guide/index.html#text=_mm...) |

## このプロジェクトでの用途

- 使用場所
- 目的

## 注意点

- よくある間違い
- 類似操作との違い

## 関連項目

- [比較する](../operations/compare.md)
````

## 書き方の方針

- トップページではデータ型を細かく分けすぎない
- intrinsic 名より先に、日本語で目的を説明する
- 最初にレーンの入力と結果を示す
- 完全な公式仕様を転載せず、公式リンクへ誘導する
- プロジェクト内の用途を必ず 1 つ以上書く
- 課題ページでは完成コードを直接書かない
